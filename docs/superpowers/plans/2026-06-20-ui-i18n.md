# UI i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add JSON-backed Russian and English localization for the Big_echo frontend UI.

**Architecture:** Add a small `src/i18n` module with JSON locale files, a React provider, a `useI18n()` hook, and Ant Design locale bridging. Keep Russian as the default language and persist the selected UI language in `localStorage`.

**Tech Stack:** React 18, Vite, TypeScript, Vitest, Ant Design 5 JSON locale files.

---

## File Structure

- Create `src/i18n/locales/ru.json`: Russian UI strings.
- Create `src/i18n/locales/en.json`: English UI strings with the same key shape.
- Create `src/i18n/index.tsx`: language type, storage key, translator, provider, hook, and Ant Design locale export.
- Create `src/i18n/index.test.tsx`: core i18n tests.
- Modify `src/AppRoot.tsx`: wrap app in `I18nProvider` and pass Ant Design locale to `ConfigProvider`.
- Modify `src/components/settings/GeneralSettings.tsx`: add language selector and translate visible labels.
- Modify frontend UI components under `src/pages`, `src/components`, and `src/lib` that render user-visible labels, placeholders, titles, aria labels, validation messages, and frontend status text.
- Modify affected tests to assert localized labels through the default Russian locale or explicit English switching.

```tsx
it("uses Russian by default and stores valid language changes", async () => {
  window.localStorage.clear();
  render(<Probe />);
  expect(screen.getByText("Настройки")).toBeInTheDocument();
  await userEvent.click(screen.getByRole("button", { name: "English" }));
  expect(screen.getByText("Settings")).toBeInTheDocument();
  expect(window.localStorage.getItem(I18N_LANGUAGE_STORAGE_KEY)).toBe("en");
});

it("falls back to Russian for invalid stored language", () => {
  window.localStorage.setItem(I18N_LANGUAGE_STORAGE_KEY, "de");
  render(<Probe />);
  expect(screen.getByText("Настройки")).toBeInTheDocument();
});

it("interpolates parameters and keeps matching locale key paths", () => {
```
